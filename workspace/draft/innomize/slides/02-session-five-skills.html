Problem Framing

Turn vague into specific

Vague Specific
"Make this better" "Reduce response time from 4 hours to 1 hour"
"Write a post" "Write a LinkedIn post announcing X, for audience Y, with tone Z"
"Summarize this" "Extract 5 key points, focus on action items, max 200 words"

Exercise

Take a vague request from your work. Rewrite it as a specific prompt.

Decomposition

Split work into verifiable steps

Bad

"Write a quarterly report"

Too big. AI will guess and get it wrong.

Good

  • Gather data from sources A, B, C
  • Verify numbers are correct
  • Analyze trends (what changed, why)
  • Draft summary (max 1 page)
  • Format for stakeholders

Key Point

Each step can be verified before moving to next. Errors caught early.

7
INNOMIZE

Decomposition Practice

Take a complex task you do regularly.

Exercise

Break it into 5-7 verifiable steps. Each step should have a clear "done" state that you can check before moving on.

Example tasks:

  • Creating a new employee onboarding package
  • Preparing monthly financial summary
  • Writing technical documentation
  • Drafting a client proposal

Evaluation

Evaluation

Know what "done" looks like before you start

Before prompting AI, ask:

  • What must be present in the output?
  • What would disqualify it?
  • What quality level is required?
  • How will I verify it's correct?

Without Evaluation Criteria

You accept "plausible-looking" output. Plausible ≠ correct.

10
INNOMIZE

Evaluation Practice

Exercise

Pick an AI task you do regularly. Write 3-5 evaluation criteria.

Example: Job Description Draft

  • ✓ Includes all required skills from manager's list
  • ✓ Matches our company tone (professional but approachable)
  • ✓ No prohibited terms (check legal list)
  • ✓ Salary range is correct
  • ✓ Benefits listed accurately

Correction

Correction

Fix the 20%, don't regenerate

When AI output is 80% right:

❌ Regenerate

  • Try different prompt
  • Hope for better
  • Waste time
  • Never improve

✓ Correct

  • Edit the 20% that's wrong
  • Build understanding
  • Learn what to fix
  • Improve next prompt

Key Insight

Teams that regenerate instead of correct waste enormous time and never improve.

Systems Thinking

Systems Thinking

Make it easier next time

After each AI task, ask:

  • Can I save this prompt for reuse?
  • Can I create a template?
  • What worked that I should repeat?
  • What failed that I should avoid?

The Difference

Teams with systems thinking improve over time. Teams without do the same work repeatedly.

15
INNOMIZE

Building Assets, Not One-Off Work

One-Off Work

  • Write prompt from scratch each time
  • No documentation
  • Knowledge stays in your head
  • Start over when you forget

System Assets

  • Prompt library (saved, organized)
  • Templates for common tasks
  • Documented what works
  • Team can learn from each other

Action

Identify one prompt you use often. Save it somewhere you can find it again.
