The Key Point

Assume anything you paste into AI could be seen by others.

Even with "privacy settings" and "enterprise plans":

  • Data leaves your computer
  • Data is processed on external servers
  • Policies can change
  • Breaches happen

Simple Rule

If you wouldn't email it to a stranger, don't paste it into AI.

What Never Enters AI

Absolute boundaries

4
INNOMIZE

Absolute Boundaries

Never paste into AI:

  • Client data: Source code, credentials, API keys, database contents
  • Personal data: Employee records, salaries, health info, ID numbers
  • Financial data: Bank accounts, specific transactions, financial reports
  • Confidential: NDAs, contracts, strategic plans, pricing
  • Credentials: Passwords, tokens, access keys

These are non-negotiable. No exceptions.

For HR & Admin

Never Enter

  • Employee personal information
  • Salary or compensation data
  • Performance reviews with names
  • Health or leave records
  • ID numbers, addresses

Safe to Use

  • Generic templates
  • Policy format questions
  • Anonymized scenarios
  • General HR practices
  • Document formatting

For Finance

Never Enter

  • Bank account numbers
  • Specific transaction data
  • Client financial details
  • Internal reports with real numbers
  • Tax filings, audit data

Safe to Use

  • Formula explanations
  • Report structure templates
  • Generic analysis methods
  • Formatting questions
  • Concept explanations

For IT

Never Enter

  • Client source code (without permission)
  • Credentials or secrets
  • Production data
  • Security configurations
  • Internal system details

Safe to Use

  • Generic code questions
  • Public documentation queries
  • Framework/library questions
  • Algorithm explanations
  • Best practice questions

Check First

Check client contract for AI usage permissions before using AI on client projects.

What to do instead

Safe Alternatives

Anonymization

  • Replace names with "Employee A", "Client B"
  • Replace specific numbers with ranges
  • Remove identifying details
  • Use percentages instead of actuals

Ask for Patterns

  • Instead of pasting data: "How do I analyze employee turnover?"
  • Instead of pasting code: "How do I implement authentication in Node.js?"
  • Ask about methods, not specific data

The Pattern

Use AI for the HOW. Add your real data manually afterward.

Quick Decision Guide

Before pasting anything into AI, ask:

  • Would I be comfortable if this appeared on a public website? → OK
  • Does this contain anyone's personal information? → DON'T
  • Is this client data or code? → DON'T (unless explicit permission)
  • Could this harm the company if leaked? → DON'T
  • Can I anonymize this first? → Anonymize, then OK
When in doubt, don't paste.

Key Takeaways

1. Assume AI data is not private

Anything you paste could potentially be seen by others.

2. Know your absolute boundaries

Personal data, credentials, client data, financials — never enter these.

3. Anonymize or use patterns

Ask AI for methods and templates. Add real data manually.

4. When in doubt, don't paste

It's better to do something manually than to risk a data breach.
