Everyone's debating whether AI is a bubble.

They're using the wrong metaphor for the real problem.

Bubble = prices inflated beyond value. Eventually pops.
Snowball = compounding effect that grows over time.

AI investment might be a bubble. 53% of fund managers think so. Even Sam Altman admits investors are "overexcited."

But here's what concerns me more:

Vibe coding is a snowball.

The numbers are alarming:

→ 40% of AI-generated code has security vulnerabilities
→ 36% of vibe coders skip testing entirely
→ Senior engineers report "development hell" reviewing AI-generated code

Each shortcut compounds. Each unreviewed function adds debt. Each skipped test creates a hole.

No-code and low-code had guardrails. Defined scope. Clear limits.

Vibe coding produces code that looks professional but lacks architectural integrity.

The bubble might pop and hurt portfolios.
The snowball keeps rolling and crushes codebases.

Which one should we worry about more?
